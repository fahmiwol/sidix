## Konteks

Apa saja endpoint API untuk NPC Registry di Mighantect?

## Pengetahuan

NPC Registry endpoints (ditambah Sprint 9A, gateway.js): GET /api/npc — list semua NPC (toJSON array + count). GET /api/npc/:id — detail satu NPC. PUT /api/npc/:id/brain — update LLM provider/model/temperature (tanpa restart, emit npc:updated socket). PUT /api/npc/:id/emotion — update state/mood/energy (emit npc:emotion socket). POST /api/npc/:id/memory — tambah memory entry {type: shortTerm|longTerm|skills, content}. POST /api/npc/:id/earning — catat earning {amount, source} ke revenue.earningsLog + update walletBalance (emit npc:earning socket). GET /api/npc/:id/prompt — dapatkan generated system prompt dari identity+emotion+memory. NpcRegistry diinit saat gateway start via loadAll(). File: server/npc-module.js + server/npc-registry.js + config/npc-profiles/48 NPC JSON files. Semua di d:\Mighan\ (workspace root).
