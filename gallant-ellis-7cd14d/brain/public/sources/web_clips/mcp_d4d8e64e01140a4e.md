## Konteks

Bagaimana cara mengimplementasikan async fire-and-forget workflow execution di Node.js ES5 dengan real LLM dispatch per step?

## Pengetahuan

Pattern di Mighantect workflow-engine.js Sprint 9G: (1) `run()` function return runId IMMEDIATELY setelah persist initial state + emit `workflow:started` socket event ā€” tidak tunggu execution; (2) `_executeAsync(run, def, engine)` dipanggil fire-and-forget (tanpa await/return); (3) `_executeAsync` pakai recursive `_next(i)` pattern (bukan loop) agar tiap step menunggu Promise resolve sebelum lanjut; (4) `_executeStepAsync` per step type: `agent` ā†’ `engine._aiConnector.callWithFallback(systemPrompt, input, opts)` yang return plain string; `service` ā†’ stub; `notify` ā†’ `io.emit`; `wait` ā†’ `_sleep(ms)` Promise; (5) Per-agent model routing: inject `opts.provider/model/max_tokens` dari `engine._agentSettings.getAgentSettings(agentId)` sebelum LLM call; (6) Error handling: step error ā†’ set `run.status='failed'`, persist, emit done ā€” tidak continue; (7) Persist setiap step dengan `_atomicWrite` (tmpā†’rename); (8) Socket events: `workflow:started`, `workflow:step` (per step done + output snippet), `workflow:done` (final), `workflow:notify`. Key: `callWithFallback` returns plain string, bukan object ā€” handle: `var output = typeof llmResult === 'string' ? llmResult : llmResult.content || String(llmResult)`.
