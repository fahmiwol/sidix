## Konteks

Sprint 9C: Upgrade tab Karyawan di MighanAdmin agar menampilkan NPC profile data (dari Sprint 9A npc-registry). Bagaimana arsitektur UI-nya?

## Pengetahuan

Sprint 9C — Admin Tab Karyawan v2:

DUAL-SOURCE FETCH:
- loadRoster() fetch parallel: GET /api/agent-ledger + GET /api/npc
- selectRoster(id) fetch parallel: GET /api/agent-ledger/:id + GET /api/npc/:id
- Backward compat: jika NPC profile tidak ada → tampil "belum ada profile" + tetap show ledger

KPI ROW (4 card):
- Total NPC (dari /api/npc count)
- Model Aktif (distinct brain.model count)
- Total Wallet coin (sum semua revenue.walletBalance)
- Dominant emotion state

LIST PANEL ENHANCEMENT:
- Avatar circle: initials dari nama, warna konsisten per-id (6 warna dari hash)
- Emotion badge inline (happy/neutral/focused/tired/stressed/excited/bored)
- Wallet balance inline (coin ⬡)
- NPC name + role dari identity

DETAIL PANEL SECTIONS:
1. Header: big avatar + name + role + emotion badge
2. Identity: personality + backstory (truncated 120 char)
3. Emotion: state badge + mood + energy bar (color-coded: green>60, yellow>30, red≤30) + quick-edit form
4. Brain: provider + model (cyan) + temperature + quick-edit form (select provider + text model + number temp)
5. Memory: shortTerm/longTerm count + skills list (max 8) + add form (type select + content input)
6. Revenue: wallet balance (yellow ⬡) + earnings count + last earning + record form
7. Ledger: existing editable textarea → PUT /api/agent-ledger/:id

4 QUICK-SAVE FUNCTIONS:
- saveNpcBrain() → PUT /api/npc/:id/brain
- saveNpcEmotion() → PUT /api/npc/:id/emotion
- saveNpcMemory() → POST /api/npc/:id/memory
- saveNpcEarning() → POST /api/npc/:id/earning
