## Konteks

Bagaimana arsitektur SIDIX agar bisa tumbuh sendiri seperti akal manusia, dari VPS tunggal hingga jaringan terdesentralisasi?

## Pengetahuan

SIDIX Self-Evolving Architecture (dari riset April 2026):

4 PILAR UTAMA:
1. Decentralized training (DiLoCo/INTELLECT-2) — 500x lebih sedikit komunikasi
2. Model merging (DARE+TIES via mergekit) — "reproduksi sel" tanpa server pusat
3. Stateful agent loop (Letta/MemGPT + Voyager skill library) — memori persisten
4. P2P coordination (libp2p + IPFS) — storage bobot terdesentralisasi

TEKNIK SELF-IMPROVEMENT:
- SPIN (Self-Play Fine-Tuning): model vs dirinya sendiri, DPO-style
- Self-Rewarding: LLM-as-Judge menilai output sendiri
- STaR: bootstrap reasoning dari chain-of-thought yang benar
- Reflexion: belajar dari kegagalan via memori verbal
- Voyager Skill Library: simpan kode yang berhasil → reuse → tumbuh

ROADMAP:
- Tahap 0 (sekarang): Qwen2.5-7B + LoRA + BM25 RAG — VPS tunggal
- Tahap 1: SPIN + Flower federated LoRA merge (2-4 node)
- Tahap 2: DiLoCo outer-loop + P2P libp2p + Byzantine Krum (10-50 node)
- Tahap 3: 30-70B model + CREATOR self-tool-creation + subnet Bittensor opsional

TARGET: bukan setara GPT-5, tapi "pertama yang terus tumbuh setelah deployment"
