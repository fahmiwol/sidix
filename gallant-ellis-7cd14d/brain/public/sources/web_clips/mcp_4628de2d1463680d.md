## Konteks

Bagaimana cara membuat SIDIX benar-benar belajar dari corpus, bukan hanya menyimpan dokumen untuk di-search?

## Pengetahuan

SIDIX menggunakan Knowledge Absorption Pipeline via corpus_to_training.py:

1. Corpus docs (Markdown) → ekstrak konsep via 5 template: definition, howto, comparison, concept, practical
2. Generate Q&A pairs ChatML format per dokumen (max 8 pairs/doc)
3. Simpan ke .data/training_generated/corpus_training_YYYY-MM-DD.jsonl
4. Upload ke Kaggle dataset → fine-tune Qwen2.5-7B + LoRA
5. Download adapter → model lebih pintar

Pipeline otomatis: setiap initiative.run_initiative_cycle() → fetch Wikipedia → reindex → auto-convert ke training pairs.

Hasil pertama: 548 training pairs dari 70 dokumen (MIGHAN 210, HAYFAR 200, FACH 130, TOARD 8).

Endpoints: GET /training/stats, GET /training/files, POST /training/run, GET /training/kaggle-guide
