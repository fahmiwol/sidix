## Pengetahuan

Sprint 9B — WorkflowEngine (server/workflow-engine.js):

ARSITEKTUR:
- ES5 singleton object WorkflowEngine (bukan class)
- Definitions: scan config/workflows/*.json → _defs cache
- Runs: in-memory _runs + atomic persist ke server/.data/workflow-runs/*.json
- Socket.io notify: WorkflowEngine.init({io}) → emit workflow:notify + workflow:done

STEP TYPES:
- agent: placeholder (Sprint 9C akan wire real dispatch ke AgentBrain)
- service: placeholder (Sprint 9C wire ke gateway service calls)
- notify: langsung emit Socket.io event + resolve {{varName}} placeholders
- wait: placeholder sync skip (async timing Sprint 9C+)

INPUT RESOLUTION: {{varName}} di step.input diganti dari run.context

5 WORKFLOW DEFS (config/workflows/):
1. microstock-daily.json — Iris prompt → autonomy cycle → notify Fahmi
2. agency-brief.json — Sari strategy → Rina platform → Dewi visual → copy → backlog
3. content-calendar.json — weekly plan → trends → captions → visuals → backlog
4. npc-morning-brief.json — Iris status → Sari brief → Budi tech → Profesor research
5. sidix-training.json — approved ideas → scoring → SIDIX capture

6 GATEWAY ENDPOINTS (/api/workflow-engine/*):
- GET /defs — list all loaded defs (re-scan setiap call)
- GET /defs/:id — single def
- POST /run — { defId, context } → returns { ok, runId, status, log }
- GET /runs — list runs (filter ?status=&defId=)
- GET /runs/:runId — run detail
- POST /runs/:runId/cancel — cancel running run

RELASI: Pelengkap agent-autonomy.js workflowTemplates (hardcoded), bukan pengganti. WorkflowEngine = JSON-defined, loadable tanpa restart.
