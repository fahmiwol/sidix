# admin panel monitoring dashboard Chart.js

**Sumber:** claude-code  
**Tanggal:** 2026-04-16  
**Tags:** monitoring, Chart.js, events, admin-panel, sprint-9H  

## Konteks

Bagaimana membuat monitoring dashboard real-time di admin panel dengan Chart.js untuk events per agent?

## Pengetahuan

Pattern Sprint 9H: (1) KPI row: total events, active agents (event dalam 1h terakhir), error count, running workflows; (2) Bar chart agents: `Object.entries(counts).sort((a,b)=>b[1]-a[1]).slice(0,15)` → Chart.js bar dengan per-bar color array; (3) Timeline chart: bucket events ke 30-min slots selama 6 jam → `Date.now() - (slots-i)*1800000` boundary, Chart.js line fill; (4) Events feed: filter client-side (agentId includes, type ===, ok===false), render 200 terbaru; (5) Per-agent summary cards: count, recentActivity (1h), minsAgo, errCount, activity bar width % = recent/total*100; (6) Auto-refresh: `setInterval(loadMonitor, 15000)` tapi clear jika tab tidak active; (7) Socket.IO `workflow:done/started/step` events juga trigger `loadMonitor()` jika tab aktif. Destroy old Chart instance sebelum create baru: `if (_monChartAgents) { _monChartAgents.destroy(); _monChartAgents = null; }`
