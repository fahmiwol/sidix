## 7. Relationship Cardinality Summary

| From | Cardinality | To | Description |
|------|-------------|-----|-------------|
| USER | 1───* | SESSION | One user, many sessions |
| SESSION | 1───* | REACT_STEP | One session, many steps |
| SESSION | 1───* | CITATION | One session, many citations |
| REACT_STEP | M───1 | TOOL | Many steps invoke same tool |
| CITATION | *───1 | SOURCE_CHUNK | Citations reference chunks (kalau type=corpus) |
| SOURCE_CHUNK | *───1 | CORPUS_NOTE | Note may have multiple chunks |
| CORPUS_NOTE | *───1 | HAFIDZ_LEDGER | (planned) Ledger entry per note |
| USER | 1───* | ACTIVITY_LOG | One user, many activities |
| CLASSROOM_PAIR | * → * | LoRA training | Pairs feed retrain |
