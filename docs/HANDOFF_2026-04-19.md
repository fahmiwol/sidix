### Plan C — Sub-Agent Internal SIDIX
Bukan satu agent monolitik, tapi orkestra agent specialized:

| Sub-Agent | Tugas | Fitur kunci |
|-----------|-------|-------------|
| **Learning Agent** | Mining data dari learning_sources, masuk corpus | Cron + dedupe + quality filter |
| **Promo Agent** | Generate konten + post ke multi-channel | Pakai content_designer + social/multi_channel_publisher |
| **Dev Agent** | Self-improvement: detect bug, propose patch | Read commit log + audit log → suggest PR |
| **Research Agent** | Existing autonomous_researcher | Integrasi dengan curriculum |
| **QA Agent** | Validasi setiap output sebelum publish | Maqashid filter + Paul-Elder check + epistemic validator |
| **Curator Agent** | Approve/reject draft, manage corpus quality | Read draft + run quality pipeline |
| **Outreach Agent** | Reply mention, engage komunitas | Pakai threads_oauth + sentiment analysis |
| **Insight Agent** | Daily/weekly report ke owner | Aggregate growth_stats + audit_log + GA4 |

**Architecture pattern**:
- Setiap sub-agent = subclass dari `BaseAgent` dengan `name`, `mandate`, `tools`, `cron_schedule`
- Communication via shared queue (`apps/brain_qa/.data/agent_inbox/`)
- Coordinator (`agent_coordinator.py`) yang routing message + monitor health
- Fallback: kalau sub-agent down, coordinator auto-restart

**Quick win (next session)**: build `apps/brain_qa/brain_qa/sub_agents/`
package dengan 2 agent dulu: `LearningAgent` (harvest from sources) +
`PromoAgent` (auto-publish). Tambah lainnya iteratif.
