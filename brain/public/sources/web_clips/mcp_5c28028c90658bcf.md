## Konteks

Apa urutan prioritas development SIDIX setelah M5 (dataset 713 samples siap)?

## Pengetahuan

Keputusan resmi Fahmi (2026-04-16):

URUTAN DEVELOPMENT SIDIX — POST M5:

1. AGENT RUNTIME DULU (C)
   - Tool-use loop (ReAct pattern)
   - Integrasi dengan brain_qa yang sudah ada
   - Permission gate (tools off by default, audit log tiap call)
   - Sesuai prinsip 09_architectural_principles.md

2. SIDIX INFERENCE ENGINE — FastAPI native (B)
   - SKIP Ollama (tidak mau pakai infrastruktur orang lain yang bisa dibuat sendiri)
   - FastAPI wrapper di atas transformers / llama.cpp
   - Ini yang jadi SIDIX Inference Engine — milik sendiri
   - Serve lokal, bisa hybrid provider

3. FINE-TUNE DI KAGGLE (A)
   - Platform: Kaggle (paling banyak dipakai, paling terpercaya, 30h/minggu gratis, P100/T4 16GB)
   - Script sudah siap: brain/datasets/sidix_finetune_colab.py
   - Dataset sudah siap: brain/datasets/finetune_sft.jsonl (713 samples)
   - QLoRA r=16, 3 epoch, ChatML format Qwen2.5-7B

FILOSOFI KEPUTUSAN:
- Yang kita bangun sendiri: agent runtime, inference engine, model weights, training data, alignment (IHOS)
- Yang kita "pinjam" (infrastruktur): GPU compute (Kaggle) — seperti Anthropic pakai AWS
- Ollama = dicoret, diganti FastAPI native SIDIX Inference Engine
