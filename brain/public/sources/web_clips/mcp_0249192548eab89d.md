## Konteks

Apa saja 7 stage dalam autonomous learning pipeline SIDIX dan bagaimana masing-masing stage diimplementasikan secara teknis?

## Pengetahuan

7-STAGE PIPELINE SIDIX (berjalan tanpa henti):

[1] SENSE — Polling sensor aktif: arXiv (6j), GitHub (12j), News (realtime), user feedback, benchmark
Output: raw data stream ke antrian ingest

[2] INGEST — Parse Markdown/PDF/HTML/code, ekstrak entitas+relasi+citation, simpan Qdrant + BM25, deduplikasi via content hash

[3] REASON — Bangun KnowledgeNode, update knowledge graph (tambah/revisi/flag kontradiksi), generate Q&A pairs via corpus_to_training.py (5 template: definition, howto, comparison, concept, practical)

[4] EVALUATE — Self-reward scoring (1-5), benchmark check (MMLU/HumanEval/MATH), gap detection per domain (52 domain total)

[5] DECIDE — Prioritaskan gap kritis, pilih action: fetch_more_data | run_spin | merge_lora | create_tool, buat curriculum

[6] ACT — Eksekusi:
- fetch_more_data → Wikipedia/arXiv fetcher (initiative.py)
- run_spin → SPIN training job di Kaggle atau lokal
- create_tool → generate Python function → simpan Voyager skill library

[7] REFLECT — Update LIVING_LOG.md, revisi confidence scores, capture ke SIDIX knowledge MCP, set agenda cycle berikutnya

Loop kembali ke [1] SENSE

STATUS TAHAP 0 SEKARANG:
