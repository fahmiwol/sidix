## ЁЯза The IHOS Foundation

**IHOS** (*Islamic Holistic Ontological System*) is not a religious constraint. It is an **epistemological architecture** тАФ a framework for how knowledge should be structured, validated, and used.

It was derived from observing one of the oldest self-expanding knowledge systems ever recorded.

### The Blueprint: A 1,400-Year-Old Cognitive System

Consider this: a single text of ~77,000 words has generated **14 centuries of derivative scholarship** тАФ yielding jurisprudence, medicine, cosmology, mathematics, linguistics, ethics, and governance. Different readers, in different times and places, derive entirely different тАФ yet internally consistent тАФ bodies of knowledge from the same source.

This is not a coincidence of literary richness. It is a **designed architecture**.

The classical Islamic scholars identified it precisely:

| Qur'anic Layer | Technical Analog | SIDIX Implementation |
|---|---|---|
| **Zahir** тАФ the explicit text | Frozen Foundation Model | `Qwen2.5-7B + LoRA` тАФ immutable base weights |
| **Batin** тАФ the latent meaning | Latent space / embeddings | `BM25 + vector corpus` тАФ contextual retrieval |
| **Asbabun Nuzul** тАФ grounded context | Grounded Generation | Every output grounded in `brand_brief + user_state + platform_context` |
| **Sanad** тАФ the chain of transmission | Provenance tracking | `[FACT] / [OPINION] / [UNKNOWN]` labels + citation chain |
| **Maqashid** тАФ the higher objectives | Objective function | 5-axis filter: life ┬╖ intellect ┬╖ faith ┬╖ lineage ┬╖ wealth |
| **Ijtihad** тАФ reasoned interpretation | Agentic reasoning | `agent_react.py` ReAct loop |
| **Naskh** тАФ abrogation/update | Knowledge conflict resolution | `naskh_handler.py` тАФ sanad-tier based supersession |
| **Tafakkur** тАФ deliberate reflection | Meta-cognition | `muhasabah_loop.py` тАФ NiyahтЖТAmalтЖТMuhasabah self-refinement |
| **Tadrij** тАФ progressive revelation | Curriculum learning | `curriculum_engine.py` L0тЖТL4 knowledge ladder |

> The key insight: Al-Qur'an's power doesn't come from **size** тАФ it comes from **architecture**.
> A frozen core. Context-sensitive derivation. Verified transmission chains. Purpose-aligned interpretation.
> This is what SIDIX translates into code.

### The 5 Principles That Follow

```
1. FROZEN CORE, LIVING EDGES
   The base model doesn't retrain arbitrarily тАФ it grows through structured LoRA adapters.
   Like a text that never changes, but whose understanding deepens with the reader.
```

### Raudah Protocol тАФ Multi-Agent Parallel Orchestration

New in v0.6: **Raudah** (╪▒┘И╪╢╪й ╪з┘Д┘Е╪╣╪▒┘Б╪й тАФ *Garden of Knowledge*) тАФ a multi-agent parallel system where specialists work concurrently and the Orchestrator synthesizes a consensus answer.

```
Task тЖТ RaudahOrchestrator.urai_task()
     тЖТ IHOS Guardrail (Maqashid check)
     тЖТ asyncio.gather([Researcher, Analyst, Writer, Engineer, Verifier])
     тЖТ RaudahOrchestrator.agregasi()   тЖР Ijma' (consensus synthesis)
     тЖТ RaudahResult.jawaban_final
```

Unlike "swarm" architectures: no vendor API, IHOS guardrail before spawn, Sanad Validator per output.

```python
import asyncio
from brain.raudah.core import run_raudah

result = asyncio.run(run_raudah("Research 5 productive waqf models in Southeast Asia"))
print(result.jawaban_final)   # synthesized consensus
print(result.durasi_s)        # e.g. 45.2s on RTX 3060
```
